# Worker Time Processing Profiles

## What They Are

Worker Time Processing Profiles are the policy bundles that govern what happens after time is captured.

## What They Control

- Repeating Time Periods
- Optional Overtime Period
- Time Consumer Sets
- Validation and calculation rule assignment context
- Approval and transfer behavior context
- Audit options

## Mental Model

If the entry profile controls how time gets in, the processing profile controls what happens after it gets in.

## Common Mistakes

- Wrong repeating period
- Wrong consumer set
- Rule assignment mismatch
- Processing profile not aligned with entry design
